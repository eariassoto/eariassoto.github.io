Unit Tests

October 21, 2017

In a previous post, I showed you a CMake template you can use for your C/C++ project. I felt that the template needed a testing framework. Since the template targets smalls projects, I think unit testing project template.

I will assume that you are familiarized with unit testing methods. In case you do not know about the Google Test framework, here is a basic introduction to this library. This post will be more of a tutorial rather than a technical post. My plan is to write about test driven development (TDD) in the future.

Requirements

I will show you how to install the Google Test in my CMake template project. The template is here, and the post where I go step by step over it is here. Feel free to read it and then come back here. If you have your own CMake project, you can follow the steps and install Google Test on your own. You only need CMake and a C/C++ compiler, which I assume you should already have.

The tests/ folder will have our unit testing project by adding the tests/CMakeLists.txt file. The tests/CMakeLists.txt.in will help tests/CMakeLists.txt include the Google Test library.

Setting up CMake

In our CMakeLists.txt file we only need to add this line:

The add_subdirectory command will add the tests project. Let’s go over the tests/CMakeLists.txt file:

There are several ways I could have included the Google Test library in the project. I could have use the find_package command, but the library needs to be installed in the computer. Another option would have been to manually copy the library in my project and compile it there. Luckily, I found a cleaner and more portable way to install the library. We will tell CMake to automatically clone and compile the library when it builds the project.

The line 5 of tests/CMakeLists.txt is actually creating a new submodule. It is copying tests/CMakeLists.txt.in to a new sub folder. This file has the data to load Google Tests as an external project:

Basically, this file just tells CMake where to download the project. Lines 8-16 of tests/CMakeLists.txt trigger the commands to build the library and add it as a submodule. The linking of the library is made in lines 40-43/ Notice that this library will only be linked to the unit testing project’s main code.

The rest of the tests/CMakeLists.txt file should be familiar to your. Similarly to our main project, your can keep your headers in tests/include and your source code in tests/src. It is important that you also include the headers and source files of your main project. Otherwise, your tests will fail to compile.

The main file of our unit testing project will search for tests and run them:

We do not need additional commands to compile our unit testing project. After you build the main project, the executable for the unit tests will be under the bin/ folder. The executable is named as the main project, but it will have the -ut suffix.
